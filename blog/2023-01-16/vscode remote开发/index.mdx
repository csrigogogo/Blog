## 前言

学习 docker 的过程中, 想到借助 docker 的虚拟机制 是否可以搭建一个干净的常用的开发镜像 , 这样协同开发时就不用担心环境不同而出现问题

<!--truncate-->

好处是什么

- 环境统一

- 开发环境配置 和 本机环境分开

- 新人易上手, 环境一致不需要单独配置

- 在任何机器和位置都能访问开发环境


## vscode remote

由于常用的开发编辑器是 vscode , 这里介绍一下 vscode 的 remote development 功能

允许使用容器, 远程服务器 , wsl 等作为开发环境

vscode remote 开发 依赖以下技术

- Remote-SSH
- Dev Containers
- WSL
